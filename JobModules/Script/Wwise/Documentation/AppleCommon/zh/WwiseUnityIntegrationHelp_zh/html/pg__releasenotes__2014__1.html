版本说明 - Wwise Unity Integration 2014.1

此次发布为 Wwise 2014.1 提供了 Wwise SDK 更新，针对错误进行了修复，提供了一个新的 Demonstration 场景，为那些使用 Wwise Unity Integration 2013.2 创建的工程提供了迁移过程。

显著功能：

  • 稳定性修复
  • 添加了一个演示场景
  • 为那些使用 Wwise Unity Integration 2013.2 创建的工程添加了迁移过程

兼容性：

  • Wwise SDK: 2014.1
  • Unity: 4.x

支持的平台：

  • Android
  • iOS
  • Linux
    • 32 位
    • 64 位
  • Mac OS X（10.6 和更高版本）
  • PS3
  • PS4
  • PS Vita
  • Windows
    • 32 位
    • 64 位
  • Windows 8（Metro）
    • Desktop
    • 在 Intel 设备上的 Windows Store App，和 ARM 平板电脑
  • Windows Phone 8.0
  • Xbox360
  • Xbox One

新功能：

  • 添加了一个演示场景，在一个单独的包中。详情请参见 pg_wwisedemoscene。
  • 为那些使用 Wwise Unity Integration 2013.2 创建的工程添加了迁移过程。
  • Wwise Components 现在在 Wwise Unity Integration 中被 UUID 引用。这可以让您在 Wwise Project 中重命名它们，并且不会破坏与 Unity 项目的链接。
  • 添加了一些可自定义的触发器，用于发布事件和加载库。详情请见 为 Wwise Events 添加新的 Trigger 。
  • 添加了一些可自定义的回调，用于 Wwise 事件。详情请见 如何与 inspector 一起使用 AkAmbient。
  • 减少了 AkSoundEngine 插件的大小。
  • 添加了一个选项，用于创建（或销毁）WwiseGlobal 对象和 AkListener。

已解决的问题：

  • WG-25783：修复了 Callback Manager 中一个内存泄漏
  • WG-25677：修复了 Mac 上的路径问题

已知问题：

  • WG-25669：Wwise Picker 的 Auto Populate 在 Mac Editor 中无法使用。
  • WG-25733：无法使用 Windows 编辑器构建 iOS/Mac 游戏，反之亦然。
