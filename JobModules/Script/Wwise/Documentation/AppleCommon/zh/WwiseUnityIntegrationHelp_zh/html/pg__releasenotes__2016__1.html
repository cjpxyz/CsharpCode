版本说明 - Wwise Unity Integration 2016.1

此次发布为 Wwise 2016.1 提供了 Wwise SDK 更新。

兼容性：

  • Wwise SDK: 2016.1
  • Unity: Unity 5.3（Personal 或 Pro）。
    		备注： 重要提示：该集成还没有在 Unity 4 上测试过。
    平台： Wwise Unity Integration 在以下平台测试过：
    Android
    iOS
    Linux
    Mac
    PS3
    PS4
    PS Vita
    Windows
    Windows Store
    Xbox One     		Unity 5.3.4p3
    Wii U Unity 5.2.4f1
    Xbox 360 Unity 5.0.1f1

已解决的问题：

  • WG-25675 修复了：在 Wwise Picker 中添加一个 “Generate SoundBanks” 按钮来为 Wwise 工程的所有平台生成 SoundBank。
  • WG-27583 修复了：现在自动管理 Unity assets 中的 SoundBank 文件夹，为当前 Unity 目标平台指向正确的 SoundBank。
  • WG-28175 修复了：在创建新场景时，WwiseGlobal 并不是每次都自动创建。
  • WG-26011 修复了：在创建一个新场景后，启用回 Ak Audio Listener 不能正确地破坏 Default Unity Audio Listener。

重要迁移说明：

  • SWIG 源代码将不再与源码压缩包 文件一起分发了。所提供的 ZIP 源码包含所需的文件和预生成的 SWIG 接口，用于构建集成包，而无需使用 SWIG。
  • 仍然使用 Wwise 2014.1.4 及更低版本的工程不能直接升级到 2016.1。请先将您的工程更新到 2015.1.6，然后到 2016.1。

已知存在的问题：

  • WG-30021：当在 Editor 中预览场景时，DecodeBanks 功能就会失效。
  • WG-30021：DecodeBanks 功能不会自动删除解码的 SoundBank。如果更新了声音库，则应该从目标设备中删除 DecodedBanks 文件夹。
  • WG-30128：无法在 Vita 平台上将解码的 SoundBank 保存到磁盘。
  • WG-30128：无法在 WiiU 平台上将解码的 SoundBank 保存到磁盘。
