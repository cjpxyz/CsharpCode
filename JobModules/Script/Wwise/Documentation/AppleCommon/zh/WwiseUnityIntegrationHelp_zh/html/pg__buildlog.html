构建日志

警告和错误消息显示在 Unity Editor 控制台和构建日志中。构建日志文件中包含了在构建集成时发出的更详细的消息。 构建日志存储在以下位置： 

UNITY_PROJECT_ROOT\Assets\Wwise\Logs\BuildWwiseUnityIntegration.log

日志消息以以下格式显示： 

时间: 消息类型: 源代码文件名 (模块名): 行数: 消息正文

示例： 

2013-09-26 09:29:56,490: INFO: BuildWwiseUnityIntegration.py (WindowsBuilder): 91: Building: Windows (Win32, Debug) ...

一个小时后，日志文件会累积并循环到一个新文件。您可以更改循环行为，方法是从 Wwise Unity 集成源包的下列文件中配置 BuildUtil.CreateLogger() 函数： 

WwiseUnityIntegration_version_platform_Src.zip\Wwise\AkSoundEngine\Common\AkSoundEngine\Common\BuildUtil.py

想了解更多信息，请参考 Python logging.handlers 模块。

在某些平台上，IDE（例如 Visual Studio）可能生成额外的日志文件。这些额外的文件可以在同一个文件夹中找到，它们的文件名中有关键词 IDE。
